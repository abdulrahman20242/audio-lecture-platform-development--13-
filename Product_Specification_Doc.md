# وثيقة مواصفات المنتج (Product Requirements Document - PRD)
## منصة المحاضرات الصوتية (Audio Lecture Platform)

### 1. نظرة عامة على المنتج
منصة تعليمية إلكترونية متخصصة في تقديم المحاضرات الصوتية للطلاب مع نظام حماية صارم يعتمد على ربط حساب الطالب بجهاز واحد فقط (Device Binding). يتيح النظام للمسؤول (المعلم) إدارة الطلاب، المواد الدراسية، المحاضرات، والأجهزة المسجلة. يتميز النظام بالبساطة والتركيز على حماية المحتوى من المشاركة غير المصرح بها.

### 2. الأهداف
- توفير منصة آمنة وسهلة الاستخدام للطلاب للاستماع إلى المحاضرات ومتابعة المواد الدراسية.
- منع مشاركة الحسابات بين الطلاب من خلال تقييد كل حساب بجهاز واحد فقط يوافق عليه المسؤول.
- منح المعلم (المسؤول) التحكم الكامل في قبول، رفض، أو إيقاف حسابات الطلاب والأجهزة.
- تنظيم المحتوى التعليمي بطريقة هرمية: مواد دراسية (Subjects) تندرج تحتها محاضرات (Lectures).

### 3. الميزات الأساسية (Core Features)

#### أ. نظام المصادقة والأمان (Authentication & Security)
- **إنشاء حساب:** يمكن للطالب إنشاء حساب جديد (الاسم، البريد الإلكتروني، كلمة المرور، رقم الهاتف).
- **تسجيل الدخول:** عبر البريد الإلكتروني وكلمة المرور.
- **استعادة كلمة المرور:** إرسال رابط إعادة تعيين كلمة المرور عبر البريد الإلكتروني.
- **ربط الجهاز (Device Binding):** عند تسجيل دخول الطالب لأول مرة من جهاز، يتم التقاط البصمة الرقمية للجهاز (Device Fingerprint/User-Agent/Screen Info) وإرسال طلب تسجيل جهاز للمسؤول ليوافق عليه. لا يمكن استخدام الحساب من جهاز آخر.

#### ب. واجهة المعلم / المسؤول (Admin Panel)
- **إدارة الطلاب:**
  - عرض قائمة بجميع الطلاب المسجلين مع حالة حساب كل منهم.
  - قبول (Approve)، رفض (Reject)، أو إيقاف (Suspend) حسابات الطلاب.
- **إدارة الأجهزة:**
  - مراجعة الأجهزة التي يحاول الطلاب الدخول منها.
  - الموافقة على الأجهزة أو رفضها.
  - فك ارتباط جهاز بطالب للسماح له بتسجيل جهاز جديد إذا لزم الأمر.
- **إدارة المحتوى (المواد والمحاضرات):**
  - إضافة، تعديل، وحذف مواد دراسية (Subjects) وإرفاق صورة غلاف ووصف لكل مادة.
  - إضافة محاضرات صوتية (Lectures) داخل كل مادة (عنوان المحاضرة، الملف الصوتي، الوصف/الملاحظات).

#### ج. واجهة الطالب (Student Interface)
- **شاشات الحالات (Status Screens):**
  - شاشة "في انتظار الموافقة" (Pending Approval) بعد التسجيل مباشرة.
  - شاشة "الحساب موقوف/مرفوض" في حالة اتخاذ المسؤول إجراءً ضد الطالب.
  - شاشة "في انتظار موافقة الجهاز" (Device Pending) عند تسجيل الدخول من جهاز جديد لم يُوافق عليه بعد.
- **استعراض المحتوى:**
  - عرض المواد الدراسية المتاحة (Subjects) في واجهة جذابة باستخدام نظام البطاقات (Cards).
  - الدخول إلى المادة لعرض قائمة المحاضرات المتاحة.
  - الاستماع للمحاضرة الصوتية باستخدام مشغل صوتي مدمج سهل التحكم.

### 4. تدفق المستخدم (User Flow)

**تدفق الطالب (Student Flow):**
1. يقوم الطالب بإنشاء حساب جديد.
2. تظهر رسالة بأن الحساب "قيد المراجعة".
3. يوافق المسؤول على الحساب.
4. يسجل الطالب الدخول، فيقوم النظام بإرسال طلب "تسجيل جهاز جديد" للمسؤول وتظهر شاشة "في انتظار موافقة الجهاز".
5. يوافق المسؤول على الجهاز المرتبط بالطالب.
6. يدخل الطالب للصفحة الرئيسية للمنصة، يتصفح المواد، ويستمع للمحاضرات الصوتية.

**تدفق المسؤول (Admin Flow):**
1. يسجل المسؤول الدخول باستخدام بيانات اعتماد المسؤول (Admin Credentials).
2. يتوجه تلقائيًا إلى لوحة التحكم (Admin Panel).
3. يضمّن تبويب "الطلاب" لمراجعة طلبات الطلاب الجدد (قبول/رفض/إيقاف).
4. يضمّن تبويب "الأجهزة" لمراجعة طلبات الأجهزة الجديدة الخاصة بالطلاب والموافقة عليها.
5. يتوجه لتبويب "المواد" لإنشاء المواد الدراسية ورفع ملفات المحاضرات داخلها.

### 5. المتطلبات التقنية (Tech Stack)
- **الواجهة الأمامية (Frontend):** React.js (v19)
- **أداة البناء والمخدم المحلي (Build Tool):** Vite
- **لغة البرمجة الأساسية:** TypeScript
- **تنسيق الواجهات (Styling):** Tailwind CSS (v4)
- **مكتبة الأيقونات:** Lucide React
- **التنبيهات التفاعلية:** Sonner (لإشعارات Toast)
- **إدارة الحالة:** React Context API (AuthContext.tsx) مخصص لإدارة حالة المصادقة والبيانات.
- **الخلفية وقواعد البيانات (Backend API & DB):** Firebase 
  - 인증 (Firebase Auth)
  - قاعدة البيانات (Cloud Firestore)

### 6. هيكل البيانات (Firestore Data Model)

- **مجموعة `users`:**
  - `id`: المعرف الفريد للمستخدم (UID)
  - `name`: اسم الطالب
  - `email`: البريد الإلكتروني
  - `phone`: رقم الهاتف (اختياري)
  - `role`: نوع الحساب (`admin` أو `student`)
  - `status`: حالة الحساب (`pending`, `approved`, `rejected`, `suspended`)
  - `createdAt`: تاريخ إنشاء الحساب (Timestamp)

- **مجموعة `devices`** (غالباً تتواجد كـ Collection منفصلة أو Sub-collection):
  - `id`: المعرف الفريد للجهاز
  - `userId`: المعرف الفريد للطالب
  - `deviceInfo`: سلسلة نصية تحتوي على معلومات الجهاز (المتصفح، نظام التشغيل)
  - `status`: حالة الموافقة متوافقة مع قواعد الأمان (`pending`, `approved`, `rejected`)
  - `lastLogin`: طابع زمني لآخر ظهور

- **مجموعة `subjects`:**
  - `id`: معرف المادة
  - `title`: اسم المادة
  - `description`: وصف قصير
  - `imageUrl`: رابط الصورة (إن وجد)
  - `createdAt`: تاريخ الإضافة

- **مجموعة `lectures`:**
  - `id`: معرف المحاضرة
  - `subjectId`: معرف المادة التابعة لها المحاضرة
  - `title`: عنوان المحاضرة
  - `audioUrl`: رابط الملف الصوتي الخاص بالمحاضرة
  - `createdAt`: تاريخ الرفع

### 7. قواعد الأمان (Firestore Rules)
- **المسؤول (Admin):** يمتلك صلاحيات القراءة والكتابة الكاملة (Read/Write) بناءً على دور `{"role": "admin"}` المحفوظ إما عبر المستند أو (Custom Claims).
- **الطلاب (Students):** 
  - يمكنهم قراءة وتحديث مستندات `users` الخاصة بهم فقط.
  - يمكنهم إضافة وتحديث مستندات `devices` الخاصة بهم.
  - يُسمح لهم بقراءة `subjects` و `lectures` **فقط** إذا كان حساب الطالب `approved` والجهاز المستخدم مقيد كـ `approved`.
  - لا يُسمح لهم بالحذف أو التعديل على المواد أو المحاضرات.

### 8. متطلبات واجهة المستخدم وتجربة المستخدم (UI/UX)
- **دعم اللغة العربية (RTL):** الواجهة مصممة بالكامل من اليمين إلى اليسار لدعم القراءة باللغة العربية.
- **التصميم المتجاوب (Responsive Design):** تكيف كامل مع الشاشات المختلفة (هواتف محمولة، أجهزة لوحية، شاشات مكتبية).
- **طابع حديث (Modern Aesthetics):** استخدام لوحة ألوان عميقة (مثل Indigo و Slate)، التدرجات اللونية (Gradients)، تأثيرات الزجاج (Glassmorphism)، وحركات انتقالية ناعمة.
- **دلالات التحميل (Loading States):** إظهار شاشة تحميل متحركة بأيقونات نابضة لضمان عدم إملاء الشاشة بالبياض أوقات انتظار رد السيرفر.
- **تجربة خالية من التحديثات المستمرة (SPA Experience):** التطبيق مكون من صقحات تفاعلية لا تتطلب إعادة تحميل الصفحة باستمرار.
